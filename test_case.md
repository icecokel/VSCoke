# Sharing Feature Verification (Revised)

## 1. 사전 준비 (Prerequisites)

- 로컬 개발 서버 실행 (`npm run dev`)

## 2. 테스트 시나리오

### 2.1. 게임 플레이 및 ID 공유

1. **플레이**: Sky Drop 또는 Block Tower 게임에서 1점 이상 획득.
2. **공유하기**: 결과 화면에서 "공유하기" 버튼 클릭.
   - **기대 결과**:
     - 로그인/저장 미완료 시: 저장 유도.
     - 저장 완료 시: 네이티브 공유 다이얼로그(모바일/지원 브라우저) 또는 클립보드 복사(미지원 환경).
     - **중요**: 모달 없이 바로 공유 동작이 실행되어야 함.
   - **데이터 확인**:
     - 공유된 URL이나 클립보드에 복사된 링크가 `/share/{uuid}` 형식을 포함하는지 확인.
     - 점수(숫자)가 아닌 ID(UUID)가 포함되어야 함.

### 2.2. 공유 페이지 (Server-side) 확인

1. **링크 접속**: 복사한 링크로 브라우저 새 탭에서 접속.
2. **데이터 표시**:
   - 게임 제목("SKY DROP" / "BLOCK TOWER") 확인.
   - 점수 및 메달이 올바르게 표시되는지 확인.
   - 날짜 표시 확인.
3. **링크 연결**:
   - "이 게임 플레이하기" -> 해당 게임으로 이동.
   - "Go to Dashboard" -> `/game`으로 이동.

### 2.3. OpenGraph 이미지 확인

1. **이미지 카드**: 디스코드, 슬랙, 카카오톡 등에 링크를 붙여넣었을 때 이미지 카드가 정상적으로 생성되는지 확인.
   - (로컬 환경에서는 `localhost` 링크라 미리보기가 안 뜰 수 있음. `ngrok` 등을 쓰거나, 직접 `/share/{id}/opengraph-image` 경로로 접속하여 확인)
