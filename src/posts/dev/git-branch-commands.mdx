# Git 브랜치 전략: 협업을 위한 필수 명령어

Git의 진가는 **브랜치(Branch)**를 사용할 때 발휘됩니다.
독립적인 작업 공간을 만들고, 안전하게 코드를 실험하고, 동료와 협업하는 방법을 알아봅시다.

## 1. 브랜치 생성과 조회: `git branch`

### 현재 브랜치 확인

```bash
git branch
```

### 새로운 브랜치 생성

```bash
git branch feature/login
```

> **Naming Rule:** 보통 `feature/기능명`, `fix/버그명`과 같은 규칙을 사용합니다.

## 2. 브랜치 이동: `git switch` (또는 `git checkout`)

새로 만든 작업 공간(브랜치)으로 이동합니다.
`checkout`은 옛날 방식이고, 최근에는 명확한 의미의 `switch`를 권장합니다.

```bash
# 최신 방식
git switch feature/login

# 구 방식
git checkout feature/login
```

## 3. 브랜치 병합: `git merge`

작업이 끝난 브랜치를 메인 브랜치와 합칠 때 사용합니다.
보통 `main` 브랜치로 이동한 후, 작업한 브랜치를 끌어와서 합칩니다.

```bash
# 1. 메인 브랜치로 이동
git switch main

# 2. 작업한 브랜치 내용 합치기
git merge feature/login
```

> **Note:** 병합 과정에서 충돌(Conflict)이 발생할 수 있습니다. 당황하지 말고 코드를 수정한 후 다시 add, commit 하면 됩니다.

## 4. 히스토리 확인: `git log`

프로젝트의 변경 기록을 그래프 형태로 확인하면 흐름을 이해하기 쉽습니다.

```bash
# 한 줄로 그래프와 함께 보기
git log --oneline --graph --all --decorate
```

## 협업 시나리오 요약

1. `git pull origin main`: 최신 코드 받기
2. `git branch feature/new-page`: 작업 브랜치 생성
3. `git switch feature/new-page`: 브랜치 이동
4. `(작업 진행)` -> `git add` -> `git commit`
5. `git push origin feature/new-page`: 작업 내용 공유
6. (GitHub 등에서 Pull Request 생성 및 머지)

이제 브랜치를 두려워하지 말고 자유롭게 활용해보세요!
