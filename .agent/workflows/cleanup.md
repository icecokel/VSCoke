---
description: Knip을 사용하여 미사용 코드 및 의존성을 탐색하고 정리(삭제)하는 워크플로우
---

1. `npm run knip` (또는 `npx knip`) 명령어를 실행하여 미사용 항목을 검사한다.
   - (참고: 이미 `knip` 설치 및 `knip.json` 설정이 완료된 상태라고 가정한다.)

2. `knip` 결과(Output)를 분석하여, 다음 카테고리별로 항목을 정리해 사용자에게 보여준다.
   - 📦 **Unused dependencies** (사용되지 않는 패키지)
   - 📄 **Unused files** (사용되지 않는 파일)
   - 📤 **Unused exports** (사용되지 않는 코드 조각)

3. 사용자에게 **"정리를 진행하시겠습니까?"** 라고 묻는다.
   - 전체 삭제를 원하면 `Yes`, 특정 항목만 원하면 구체적으로 말해달라고 안내한다.
   - (예: "dependencies만 삭제해 줘" 또는 "src/utils.ts 파일만 지워줘")

4. 사용자 승인 시, 다음 작업을 수행한다:
   - **패키지**: `npm uninstall [패키지명]`을 실행한다.
   - **파일**: `rm [파일명]`을 실행하여 삭제한다.
   - **Exports**: 해당 라이브러리를 사용 중인 코드가 없음을 확인하고, 가능하다면 파일 내에서 해당 export 문을 주석 처리하거나 제거한다. (수정 후 `lint` 체크 권장)

5. 정리 작업 후 `npm run build`를 실행하여, 삭제로 인해 프로젝트가 깨지지 않았는지 최종 검증한다.
