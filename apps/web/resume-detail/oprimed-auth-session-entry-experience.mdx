## 개요 (Overview)

**인증·세션·제품 진입 경험 정리**는 Portal, Labs, Nexus, Trial로 이어지는 제품군의 로그인, 세션 만료, 제품 이동 경험을 통일한 작업입니다.

여러 제품이 분리되어 있어 사용자는 Portal을 통해 각 제품으로 이동하고, 제품 간 이동 중 인증 쿠키와 세션 상태가 일관되게 유지되어야 했습니다. 이 작업에서는 로그인 게이트, 세션 복원, 제품 라우팅, 모바일 접근 정책을 정리했습니다.

## 주요 구현 (Key Features)

### 1. Portal 진입 흐름 정리

- **로그인 게이트와 제품 라우팅**:
  - Portal 로그인 게이트, 인증 쿠키, 세션 복원, 제품 라우팅 흐름을 정리했습니다.
  - Labs, Nexus, Trial 등 여러 제품으로 이동하는 공통 진입점과 제품 카드 UI를 개선했습니다.
- **referrer 기반 리다이렉트**:
  - 제품에서 Portal로 이동한 뒤 다시 원래 제품으로 돌아갈 수 있도록 referrer 기반 로그인 리다이렉트 흐름을 보정했습니다.

### 2. 세션 만료 경험 통일

- **세션 만료 SDK 연결**:
  - 세션 만료 CDN SDK와 공통 모달 흐름을 연결했습니다.
  - 인증 실패 시 제품마다 다르게 보이던 사용자 경험을 공통 모달과 이동 흐름으로 통일했습니다.
- **모바일 접근 정책**:
  - 모바일 전용 안내 페이지와 비모바일 접근 차단 등 제품별 진입 정책을 정리했습니다.
  - 제품 특성에 맞지 않는 접근 환경에서는 안내 화면으로 유도하도록 보정했습니다.

### 3. 운영 안정성 보강

- **보안 헤더와 테스트**:
  - CSP 응답 헤더를 정리해 포털 보안 설정을 보강했습니다.
  - 인증/API 테스트와 Playwright 기반 검증 흐름을 추가해 운영 중 확인 가능한 기준을 만들었습니다.

## 기술적 포인트 (Technical Depth)

- **Next.js 라우팅**: 제품 라우팅과 로그인 리다이렉트 흐름을 Next.js 기준으로 정리했습니다.
- **인증 상태 관리**: 인증 쿠키, 세션 복원, 세션 만료 모달을 사용자 흐름 기준으로 연결했습니다.
- **검증 자동화**: Playwright 기반 테스트로 인증/라우팅 흐름의 회귀 가능성을 줄였습니다.

## 결과 (Outcome)

Portal, Labs, Nexus, Trial로 이어지는 제품 진입 경험을 정리해 여러 FE 제품 사이의 인증 실패, 세션 만료, 제품 이동 경험을 더 일관되게 만들었습니다.
