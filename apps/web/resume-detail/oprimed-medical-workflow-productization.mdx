## 개요 (Overview)

**의료·임상 분석 워크플로우 FE 제품화**는 의료·임상 데이터 분석 플랫폼에서 복잡한 시뮬레이션 입력, 실행, 결과 확인 흐름을 사용자가 실제로 다룰 수 있는 프론트엔드 제품 화면으로 구현한 작업입니다.

Nexus와 Trial 화면을 중심으로 분석 조건 입력, 검증, 실행, 결과 테이블, 차트, PDF/Excel 출력까지 이어지는 흐름을 정리했습니다.

## 주요 구현 (Key Features)

### 1. 분석 워크플로우 화면 구현

- **DRD, TRP, SUB, TOP, Twin Outcome 흐름 개발**:
  - 각 분석 유형의 입력, 검증, 실행, 결과 확인 단계를 화면 흐름으로 구성했습니다.
  - 분석 조건이 API 요청값과 화면 상태에 일관되게 반영되도록 입력 상태와 payload 변환을 정리했습니다.
- **도메인 데이터 연결**:
  - 환자 정보, 질병, 약물, baseline feature, 분석 조건, 시뮬레이션 결과처럼 서로 연결된 데이터를 화면 상태와 API 요청값으로 정리했습니다.
  - 질병별 입력 분기와 약물 선택 흐름에서 사용자가 선택한 조건이 결과 화면까지 이어지도록 보정했습니다.

### 2. 분석 결과 확인 기능

- **결과 테이블과 차트**:
  - 시뮬레이션 결과를 테이블과 차트로 확인할 수 있도록 화면 구조를 정리했습니다.
  - 차트 fullscreen, 결과 테이블, 리포트 배치 등 분석 결과를 검토하는 화면 기능을 구현했습니다.
- **PDF/Excel 흐름**:
  - Excel import를 통해 입력 데이터를 화면 상태로 반영하는 흐름을 구성했습니다.
  - PDF preview/download와 리포트 레이아웃을 정리해 결과를 공유 가능한 형태로 확인할 수 있게 했습니다.

### 3. 복잡한 상태 흐름 보정

- **상태 복원과 handoff**:
  - 다중 약물 선택, Drive load, IndexedDB 복원, View parameter, handoff 데이터처럼 상태 복잡도가 높은 화면 흐름을 보정했습니다.
  - 재진입 시 기존 입력값이 잘못 덮어써지거나 결과 화면과 입력 화면의 상태가 어긋나는 문제를 줄였습니다.
- **UI 완성도 개선**:
  - Figma 기준으로 테이블, 모달, 브레드크럼, 카드, 차트, PDF 레이아웃을 정렬했습니다.
  - 분석 화면의 사용성과 제품 완성도를 높이기 위해 반복되는 UI 패턴을 정리했습니다.

## 기술적 포인트 (Technical Depth)

- **상태 관리**: `Zustand`, `TanStack Query`, `React Hook Form`을 조합해 서버 데이터, 입력 상태, 검증 상태를 분리했습니다.
- **시각화**: `ECharts`, `Plotly`를 활용해 분석 결과를 차트와 fullscreen 화면으로 확인할 수 있게 했습니다.
- **리포트 출력**: `jsPDF`, `xlsx`를 활용해 PDF/Excel 기반 결과 확인 흐름을 구성했습니다.

## 결과 (Outcome)

복잡한 의료·임상 분석 워크플로우를 사용자가 입력, 실행, 결과 확인, PDF/Excel 출력까지 이어갈 수 있는 프론트엔드 제품 흐름으로 정리했습니다.
