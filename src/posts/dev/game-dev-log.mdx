# Sky Drop 게임 제작기: React와 Phaser의 만남

프로젝트의 재미 요소를 더하기 위해 시작된 **Sky Drop** 게임 개발.
React 기반의 웹 앱에 Phaser 게임 엔진을 얹으며 겪었던 치열했던 개발 기록을 정리해봅니다.

## 1. 시작: React와 Phaser의 공존 (Initial Implementation)

> **Commit `99e2dbd`**: _feat(game): 반응형 UI 및 메커니즘을 포함한 Sky Drop 게임 구현_

첫 단추는 "과연 Next.js 환경에서 Phaser가 잘 돌아갈까?"라는 의문에서 시작되었습니다.
`PhaserGame` 컴포넌트를 만들고, React의 State와 Phaser의 Event를 연결하는 구조를 잡았습니다.
초기에는 단순히 하늘에서 떨어지는 물체를 피하는 기본 메커니즘만 구현되었습니다.

## 2. 난관: 배포와 렌더링 이슈 (Troubles)

개발 과정이 순탄치만은 않았습니다. 특히 배포 환경에서의 이슈들이 발목을 잡았습니다.

### WebGL vs Canvas

> **Commit `1ec53b3`**: _fix: Phaser WebGL 비활성화 테스트 - Canvas 렌더러로 변경_

일부 환경에서 WebGL 렌더링 호환성 문제가 발생하여, 안정성을 위해 Canvas 렌더러로 전환하는 실험을 진행했습니다.

### 라우팅의 악몽

> **Commit `c41328f`, `d45086b`**: _라우트 문제 테스트 및 전역 404 페이지 추가_

Vercel 배포 후, 특정 라우트(`sky-drop` vs `skydrop`)에서 게임 리소스가 로드되지 않거나 404 에러가 발생하는 문제가 있었습니다.
이는 Next.js의 파일 기반 라우팅과 Phaser의 에셋 로딩 경로 간의 충돌이었고, 결국 폴더 구조 변경과 절대 경로 설정을 통해 해결했습니다.

## 3. 개선: 완성도를 높이는 디테일 (Polishing)

기능이 돌아가기 시작한 후, 사용성을 높이기 위한 작업들이 이어졌습니다.

### 커스텀 라우터 도입

> **Commit `1a3afb6`**: _refactor: 커스텀 라우터 전체 적용_

게임 내에서 뒤로가기를 했을 때 히스토리 관리가 꼬이는 문제를 해결하기 위해 `useCustomRouter` 훅을 도입했습니다. 이제 브라우저 히스토리와 탭 상태가 완벽하게 동기화됩니다.

### 모바일 퍼스트 & 반응형

> **Commit `b23713b`, `154fc03`**: _style: 모바일 퍼스트 레이아웃 적용_

PC에서는 문제없던 게임이 모바일 가로 모드에서는 화면이 잘리는 현상이 발생했습니다. `max-height`를 조절하고, 모바일 사이드바(`3bf4c39`)를 개선하여 쾌적한 플레이 환경을 만들었습니다.

### 시각적 효과 강화

> **Commit `b69d3f4`**: _feat: 게임 블럭 폭발 효과 개선 및 로딩 바 추가_

단조로운 게임 플레이에 타격감을 주기 위해 블록이 터질 때 파티클 효과를 추가하고, 에셋 로딩 상태를 보여주는 로딩 바를 구현했습니다.

## 4. 마무리: 글로벌 서비스로 (i18n)

> **Commit `38e1b8a`**: _feat: 전체 i18n 마이그레이션 적용_

마지막 퍼즐은 다국어 지원이었습니다. 게임 내 하드코딩된 텍스트들을 모두 뽑아내어 리소스 파일로 옮겼고, 이제 한국어와 영어를 완벽하게 지원합니다.
