### **2\. npm init 및 nodemon 설치**

프로젝트를 진행 할 디렉토리를 생성해주고, npm init을 하여  package.json 파일을 생성해 줍니다.

```
npm init
```

![](https://blog.kakaocdn.net/dna/cKvnzR/btrcM1ARRAX/AAAAAAAAAAAAAAAAAAAAACbHRMrTmsFhUEcEPQaLEpU9YDeFJfETGtfltV_oieuR/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=OyOmQc4AlumxoF39N5vr48z45Wo%3D)

저는 github에서 클론 생성한 디렉토리라 README.md 가 있습니다. 일반적으로 디렉토리를 생성했다면 없는 것이 정상입니다.

nodemon은 파일을 수정 후 재기동 없이 변경사항을 적용 시키기 위해 사용 됩니다. 자세한 정보는 링크에서 확인 부탁드립니다.

```
npm install -g nodemon
```

[https://www.npmjs.com/package/nodemon](https://www.npmjs.com/package/nodemon)

![](https://blog.kakaocdn.net/dna/wMlT5/btrcQf5G2bB/AAAAAAAAAAAAAAAAAAAAAOZsnphf_doKOWKTAn0JMToE3K1etxOsEYbCpRvt6tZW/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=o9ZOz%2BPBLhQcAPXNLZ4lrdw0hQs%3D)

환경변수 세팅이 번거로워 글로벌로 설치를 진행했습니다만 진행 도중에 환경변수 세팅하여, 진행 되었음을 알려드립니다. 환경변수 세팅에 대해서는 다음에 포스팅을 진행하겠습니다.

설치 완료 확인은, 간단한 콘솔로그를 보여주는 js 파일을 생성하여 nodemon 으로 실행 시켜보는 방법입니다. 

```
// TEST Code
// Filename : index.js or index.ts
// TEST console : nodemon . or nodemon ./index.js

setInterval(() => {
  console.log(`nodemon Test Time : ${new Date().toLocaleTimeString()}`);
}, 1000);
```

실행 시켜본 후 nodemon을 종료하지 않은 상태에서, 파일을 수정하여 종료 없이 적용이 되는지를 확인 합니다.

### **3\. typescript 및 ts-node 설치 및 세팅**

 nodemon까지 무리 없이 설치가 진행 되었다면, typescript 설치를 진행해 보겠습니다.

[https://www.npmjs.com/package/ts-node](https://www.npmjs.com/package/ts-node)

 [ts-node

TypeScript execution environment and REPL for node.js, with source map support

www.npmjs.com](https://www.npmjs.com/package/ts-node)

```
npm install -D typescript
npm install -D ts-node
```

![](https://blog.kakaocdn.net/dna/utwMp/btrcNlFR6mD/AAAAAAAAAAAAAAAAAAAAANxiVSQL-Zrzny8xdOrS-x0cwQxiQHyOxopGKOS1PCqW/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=3PHHMQvpa%2B6P7feqnrag9E13BMI%3D)

정상적인 설치가 진행 되었다면, tsc 명령어를 사용할 수 있습니다. 테스트를 하기 위해 tsc -v 로 tsc 버전을 확인해 봅니다.

```
tsc -v
```

![](https://blog.kakaocdn.net/dna/bLI9P0/btrcNNoyhg2/AAAAAAAAAAAAAAAAAAAAAAVikYlofPPg5wppxkKSklhf48XieSAsmdkGRgooF0nJ/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=yZewTpW%2Fj4uK%2FdpKSEtHTlDcCq0%3D)

만약 위와 같은 오류가 뜬다면, 따라서 진행 부탁드립니다. 정상적으로 버전 확인이 가능하다면 따라하지 않아도 됩니다.

windows 10 기준 \[시작\]버튼 을 클릭하고, (또는 \[Windows 버튼\]) "windows powershell" 이라고 쳐줍니다.

![](https://blog.kakaocdn.net/dna/q6NYs/btrcMWMH1Wy/AAAAAAAAAAAAAAAAAAAAADZ1Cv-8uaQ0aZbS7UFlr97gkUfFsNBKHytmJXjuHadW/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=cFgrZhWEH1gHal44%2F1BxuPZEmHo%3D)

Windows PowerShell이 보이면, 우 클릭하여 \[관리자 권한으로 실행\]을 눌러 창을 띄웁니다.

![](https://blog.kakaocdn.net/dna/bSwPqK/btrcQfYUVPk/AAAAAAAAAAAAAAAAAAAAAMtyxdeyUxv1eD2scU-GXNh315nJszwHFCQdWxoCKnW5/img.png?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=0IIPH9Lt2TdARXp%2BArUbYioy04U%3D)

```
Set-ExecutionPolicy Unrestricted
```

![](https://blog.kakaocdn.net/dna/zVoAm/btrcONO7vXK/AAAAAAAAAAAAAAAAAAAAAEbMubIo-2cyNW51RX4tnfSc7sg0l1mSuAdLllkGmE8G/img.png?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=LHYdgupTH9J1BPwXNHJ84BJq09Y%3D)

tsc --init 명령을 사용하여 tsconfig.json 파일을 생성합니다.

```
tsc --init
```

저는 위와 같이 세팅을 했습니다.

//  구성 파일에서 입력을 찾을 수 없습니다. 지정된 '포함' 경로는 '\[\]'이고 '제외' 경로는 '\[\]'이었습니다.

설치 후 그림과 같이 경로 문제로 에러처럼 보일 수 있으나 당황하지 않고 따라오시면 해결이 됩니다. 해당 문제를 해결하기 위해 tsconfig-paths를 설치합니다.

[https://www.npmjs.com/package/tsconfig-paths](https://www.npmjs.com/package/tsconfig-paths)

 [tsconfig-paths

Load node modules according to tsconfig paths, in run-time or via API.

www.npmjs.com](https://www.npmjs.com/package/tsconfig-paths)

```
npm install --save-dev tsconfig-paths
```

설치 후에도 오류 처럼 보일 수 있으나, 시간이 지나면 해결 될겁니다.(저는 그랬습니다.) 뭔가 속도가 느린 느낌이긴 하더라구요.

여기까지만 진행해도 됩니다만, 추가 적으로 좋은 코드를 작성하기 위해 eslint를 설치 및 세팅을 하겠습니다.

[https://www.npmjs.com/package/eslint](https://www.npmjs.com/package/eslint)

 [eslint

An AST-based pattern checker for JavaScript.

www.npmjs.com](https://www.npmjs.com/package/eslint)

```
npm install eslint --save-dev
```

### **4\. express 설치 및 세팅**

본격적으로 서버로 사용할 수 있는 express를 설치 진행 전 필수 항목은 아니나 한 가지 작업을 진행해 줍니다.

```
npm install --dev @types/node
```

![](https://blog.kakaocdn.net/dna/tjGBn/btrcStWUh1x/AAAAAAAAAAAAAAAAAAAAAEoFz30MiqiIqwPDUW00B84QPyZUnoGZdQMqxiUVYAF-/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=xTipDmeyU0%2B29SJ7ezLcz3jLIMI%3D)

사실 정확이 어떤 기능을 하는지 알 지 못합니다만, 저는 진행 했기에 권유 드립니다. 정확한 이유를 아시는 분은 댓글 부탁드립니다.

express를 설치합니다.

```
npm install express
```

![](https://blog.kakaocdn.net/dna/72ra6/btrcM09LzUQ/AAAAAAAAAAAAAAAAAAAAAEh803f6Dp7o7rYLLL6g678-sjRYkdozQ-tnFrHfFuQ3/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=uC1P5wUMhQsaZk6JIzj8gx5t%2F5U%3D)

설치가 완료 되었다면, 편한 실행을 위해 scripts를 등록합니다.

![](https://blog.kakaocdn.net/dna/lMhPK/btrcMHvRqoJ/AAAAAAAAAAAAAAAAAAAAAESIZa8Rqh_fZdONwQ5IL_vEd0dkljBd8jOwtxwfbJUb/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=WOd1CcpruMhxRGT094HIWMC5J2w%3D)

저는 서버기동으로 사용할 파일을 src/server.ts로 지정했기때문에 그림과 같이 설정했습니다. 주로 일반적으로 파일명을 index.ts 또는 app.ts 등으로 설정합니다.

src 디렉토리를 생성 한 후 src 밑에 server.ts 파일을 생성 및 작성합니다.

![](https://blog.kakaocdn.net/dna/xteVc/btrcMVG3riE/AAAAAAAAAAAAAAAAAAAAAHlBL557NviZt6j8bAJ-XMegpawYzg2_G6Ebj81Mwxvx/img.png?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=OhcNDaxWAFLYUxG7dhUoZ4dJBC8%3D)

![](https://blog.kakaocdn.net/dna/cZFxoy/btrcOezsXdY/AAAAAAAAAAAAAAAAAAAAAM34E_UnE70UXwYH1FlDTsbeSG7M-MUMYFoT1RBTRaUN/img.jpg?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1767193199&allow_ip=&allow_referer=&signature=VcncNg%2BadBH3c9q6tUzVKyylgrU%3D)

```
import express from "express";

const app = express();

app.get("/", (req, res) => {
  console.log(req);

  res.send("성공입니다 고생하셨어요!");
});

app.listen(3000);
```

작성이 끝났다면 앞서 등록해둔 scripts 인 start 명령을 이용하여, 서버를 기동 합니다.

```
npm start
```

사용하시는 브라우저를 열고 지정한 3000번 포트로 접속 시도하여 서버가 기동되었는지 확인합니다.
