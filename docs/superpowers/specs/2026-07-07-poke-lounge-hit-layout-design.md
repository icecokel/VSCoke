# Poke Lounge Hit And Layout Design

## Goal

포케라운지 전투에서 데미지를 받은 포켓몬이 즉시 피격 피드백을 보이고, 웹/모바일 모두 플레이 캔버스가 4:3 비율을 유지하며, 모바일 조작 UI가 뷰포트 범위를 넘지 않게 한다.

## Decisions

- 4:3 고정 대상은 전투/필드가 렌더링되는 `#game-root` 캔버스다.
- 모바일 조작 UI는 캔버스 아래 별도 조작 영역으로 유지하되, 화면 높이가 작으면 버튼 크기와 예약 높이를 줄여 뷰포트 안에 들어오게 한다.
- 피격 연출은 배틀 상태 변경으로 HP가 감소할 때 같은 지점에서 트리거한다. 전투 로직 결과는 바꾸지 않고 렌더링 상태만 흔든다.
- E2E에서는 transient 애니메이션을 직접 눈으로 판별하지 않고, 애니메이션 시작 카운터와 DOM rect/비율을 검증한다.

## Acceptance

- 상대 또는 플레이어 HP가 감소하면 해당 side의 피격 애니메이션 시작 카운터가 증가한다.
- `#game-root canvas`는 데스크톱/모바일/짧은 모바일 뷰포트에서 4:3 비율을 유지한다.
- 모바일 조작 버튼과 전체 문서는 작은 모바일 뷰포트에서 수평/수직 범위를 넘지 않는다.
- 기존 Poke Lounge E2E, web typecheck, web lint/build가 통과한다.
