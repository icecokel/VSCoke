## 기본 정보

- 회사: 주식회사 코드크레용
- 기간: 2025.02 - 2025.07
- 고용 형태: 정규직
- 핵심 포지션: AI 자막 번역과 콘텐츠 운영 자동화
- 기술 스택: Node.js, TypeScript, Express, AWS Bedrock, OpenAI API, Google Cloud Translation, Axios, Cheerio, Vercel Serverless Functions

## 핵심 요약

개발자가 CLI로 실행하던 자막 번역을 운영자가 파일 업로드부터 검수·다운로드까지 처리하는 웹 도구로 확장했습니다. 별도로 매일 반복하던 무단 도용 콘텐츠 검색도 3개 플랫폼을 한 화면에서 확인하는 서버 기반 도구로 전환했습니다.

## 1. 자막 번역 CLI를 반복 가능한 파이프라인으로 전환

### 문제

- 번역은 개발자가 CLI와 파일 경로를 직접 다뤄야 했고, 여러 작품과 언어를 처리할 때 같은 명령과 확인을 반복해야 했습니다.
- AI 응답의 자막 누락, 시간축 불일치와 결과 형식 오류를 사람이 마지막에 발견할 수 있었습니다.
- 모델이나 번역 언어가 바뀔 때 파일 처리와 검증 코드까지 함께 수정될 수 있었습니다.

### 변경

- VTT 자막을 읽고 SRT 입력은 WEBVTT 헤더와 타임스탬프 형식으로 변환했습니다.
- 자막 배열을 청크로 나누고, AI 응답은 `id`와 `translatedText`를 가진 고정 JSON 형태로 받게 했습니다.
- 여러 파일과 대상 언어를 batch로 묶어 병렬 요청하고, 검증에 실패한 대상은 청크 크기를 줄여 다시 요청했습니다.
- 한국어·영어 자막을 일본어 번역의 참조로 사용할지 선택할 수 있게 했습니다.
- 모델 선택 경로를 분리해 Bedrock과 OpenAI 서비스 변경이 번역 전체 흐름에 퍼지지 않게 했습니다.

## 2. 번역 결과를 파일 단위로 검증

- 원본과 번역본의 자막 개수를 비교했습니다.
- 누락되거나 비어 있는 번역 문장과 시작·종료 시간 불일치를 검사했습니다.
- 번역 결과의 고정 형식이 깨지거나 원본 자막에 매핑되지 않는 항목을 실패로 분리했습니다.
- 검수가 끝난 결과를 VTT로 직접 내려받고, 파일 이동·업로드 단계에서 다시 사용할 수 있게 했습니다.

## 3. 운영자용 웹 도구로 확장

- 콘텐츠·에피소드 정보, 원문 자막, 대상 언어와 캐릭터 정보를 단계별로 입력하게 했습니다.
- 한국어·영어 VTT를 함께 올리고 자막별 캐릭터를 체크박스로 지정할 수 있게 했습니다.
- 저장한 콘텐츠와 캐릭터 설정을 다시 불러오고 VTT·JSON 결과를 내려받게 했습니다.
- CLI 명령과 파일 경로를 알지 못해도 운영자가 같은 번역·검수 순서를 실행할 수 있게 했습니다.

### 결과

- 도입 전 3명이 하루 1~2개 작품을 처리하던 번역 업무를 도입 후 1명이 하루 6~8개 처리할 수 있게 했습니다.
- 이 도구는 퇴사 이후에도 운영 업무에 계속 사용되고 있습니다.

## 4. 무단 도용 콘텐츠 검색 자동화

### 문제

- Bilibili TV, Bilibili와 Dailymotion을 각각 검색하고 결과를 비교하는 데 매일 약 50분이 들었습니다.
- 브라우저에서 직접 검색할 때 차단과 실행 제약이 발생했습니다.

### 변경·결과

- 검색을 Express·Node.js 서버로 옮기고 3개 플랫폼을 한 화면에 배치했습니다.
- 키워드를 10개 단위로 묶어 병렬 검색하고, 제목을 정규화한 뒤 정확 일치·부분 일치·Levenshtein 유사도 기준으로 후보를 걸렀습니다.
- 플랫폼별 키워드를 저장하고 결과를 썸네일·외부 링크와 함께 묶어 확인하게 했습니다.
- Vercel Serverless Functions 배포 구조로 전환해 매일 약 50분 걸리던 확인 업무를 10분 이내로 줄였습니다.
