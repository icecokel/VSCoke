# Wordle 게임 테스트 케이스

## 1. 개요

Wordle 게임의 로직과 UI가 정상 작동하는지 확인하는 검증 절차입니다.

## 2. 테스트 환경

- **URL**: `http://localhost:3000/game/wordle` (포트 번호는 실행 환경에 따라 다름)
- **Browser**: Chrome / Safari / Edge 최신 버전

## 3. 테스트 시나리오

|    ID     | 테스트 항목             | 절차                                                                             | 예상 결과                                                                                               | 비고 |
| :-------: | :---------------------- | :------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------ | :--- |
| **TC-01** | 게임 진입               | 1. `/game/wordle` 접속 <br> 2. 화면 로딩                                         | - 30개의 빈 타일 그리드 표시 <br> - 하단 가상 키보드 표시                                               |      |
| **TC-02** | 단어 입력 (키보드)      | 1. 물리 키보드로 알파벳 입력 <br> 2. 가상 키보드 클릭으로 입력                   | - 해당 글자가 현재 행의 타일에 채워짐 <br> - 5글자 초과 입력 불가                                       |      |
| **TC-03** | 글자 삭제               | 1. `Backspace` 키 누름 <br> 2. 가상 키보드의 Backspace 클릭                      | - 마지막 글자가 지워짐                                                                                  |      |
| **TC-04** | 유효하지 않은 단어 제출 | 1. 5글자 채우지 않고 `Enter` <br> 2. 사전에 없는 단어(예: QQQQQ) 입력 후 `Enter` | - 제출되지 않음 (콘솔 로그 확인: "Not in word list" 등) <br> - 다음 줄로 넘어가지 않음                  |      |
| **TC-05** | 정답 제출 (승리)        | 1. 디버그용 정답(화면 하단) 확인 <br> 2. 정답 단어 입력 후 `Enter`               | - 타일이 모두 초록색으로 변함 <br> - "축하합니다!" 알림 표시 <br> - 더 이상 입력 불가능                 |      |
| **TC-06** | 오답 제출 및 힌트       | 1. 정답이 아닌 유효 단어 입력 후 `Enter`                                         | - 타일 색상 변경 (초록/노랑/회색) <br> - 가상 키보드 키 색상도 동일하게 변경 <br> - 다음 줄로 커서 이동 |      |
| **TC-07** | 게임 패배               | 1. 6번 시도 모두 오답 제출                                                       | - 마지막 시도 후 "아쉽네요" 알림 및 정답 공개 <br> - 더 이상 입력 불가능                                |      |
| **TC-08** | 게임 재시작             | 1. 우측 상단 새로고침 아이콘 클릭                                                | - 보드 및 키보드 초기화 <br> - 새로운 정답 단어 설정                                                    |      |

## 4. 디버깅 팁

- 개발 모드에서는 화면 최하단에 작은 글씨로 `Debug Answer: [단어]`가 표시됩니다. 이를 보고 정답 테스트를 쉽게 할 수 있습니다.
